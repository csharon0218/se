# 1. 서론

## 1.1 문서의 목적 및 범위

본 문서는 **SyncSpace(클라우드 파일 공유 시스템)**의 요구사항을 정의하기 위한 문서이다.

SyncSpace는 조직 내부 파일을 안전하게 저장하고, 빠르게 탐색하며, 권한에 따라 정확하게 공유할 수 있도록 지원하는 웹 기반 파일 공유 시스템이다. 본 문서는 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항을 명확히 정의함으로써 이후 요구사항 분석서, 소프트웨어 설계서, 구현 및 테스트 단계의 기준 문서로 활용된다.

본 문서의 범위는 다음과 같다.

① 대상 시스템의 목적과 주요 기능 정리  
② 기능적 요구사항 정의  
③ 비기능적 요구사항 정의  
④ 인터페이스 요구사항 정의  
⑤ 기타 정책적·운영적 요구사항 정리  
⑥ 유사 요구사항 간 역할 구분  
⑦ 요구사항과 유스케이스 간 추적성 정리  

즉, 본 문서는 SyncSpace가 **무엇을 해야 하는가**와 함께, **어떤 수준으로 동작해야 하는가**를 구체적으로 정의하는 문서이다. 또한 요구사항 추적성 매트릭스를 통해 각 요구사항이 어떤 유스케이스 및 이후 SRS 항목과 연결되는지 확인할 수 있도록 한다.

특히 본 문서에서는 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항이 서로 중복되지 않도록 각 요구사항의 목적을 구분하여 작성한다. 예를 들어 업로드 진행 상태 표시와 관련된 요구사항은 기능 제공 여부, 사용자 인터페이스 제공 여부, 성능 기준을 각각 구분하여 정의한다.

## 1.2 대상 시스템 개요

### 1.2.1 프로젝트 정의

SyncSpace는 조직 내부 문서, 설계 파일, 보고서, 이미지, 압축 파일, 데이터 파일 등을 안전하고 효율적으로 저장·관리·공유할 수 있도록 지원하는 **웹 기반 클라우드 파일 공유 시스템**이다.

현재 조직 내 파일은 이메일, 메신저, 개인 저장소 등에 분산되어 관리되며, 이로 인해 다음과 같은 문제가 발생할 수 있다.

- 파일 최신 버전 혼선
- 필요한 파일 탐색 시간 증가
- 외부 공유 시 보안 통제의 어려움
- 대용량 파일 공유 및 재업로드의 불편함
- 사용자별 권한 관리의 어려움

본 프로젝트는 이러한 문제를 줄이기 위해 파일의 중앙 집중 관리, 검색, 공유, 버전 관리, 권한 기반 접근 제어 기능을 제공하는 시스템 구축을 목표로 한다.

### 1.2.2 주요 기능 설명

| ID | 설명 |
|---|---|
| F01 | 파일 업로드 및 다운로드 |
| F02 | 폴더 생성, 수정, 삭제 |
| F03 | 파일 이동 및 정리 |
| F04 | 사용자 간 파일 공유 |
| F05 | 외부 링크 공유 |
| F06 | 파일 검색 |
| F07 | 파일 버전 관리 |
| F08 | 계정 및 권한 관리 |
| F09 | 보안 및 접근 통제 |
| F10 | 대용량 업무 파일 업로드 지원 |
| F11 | 업로드 재시도 및 이어올리기 지원 |
| F12 | 관리자 정책 관리 |

## 1.3 용어 정의

| 용어 | 설명 |
|---|---|
| 클라우드 파일 공유 시스템 | 네트워크를 통해 파일을 저장, 관리, 공유할 수 있는 시스템 |
| 권한 기반 접근 제어 | 사용자별 또는 역할별로 파일 접근 및 공유 범위를 제한하는 방식 |
| 외부 링크 공유 | 시스템 외부 사용자에게 URL 기반으로 파일을 공유하는 기능 |
| 버전 관리 | 파일 변경 이력을 저장하고 이전 상태로 복원할 수 있는 기능 |
| 대용량 파일 업로드 | 설계 파일, 영상 파일, 압축 파일, 데이터 파일 등 큰 용량의 파일을 업로드하는 기능 |
| 이어올리기 | 업로드 중단 시 전체를 다시 업로드하지 않고 중단된 지점부터 재전송하는 방식 |
| 업로드 진행률 | 사용자가 파일 업로드가 어느 정도 진행되었는지 확인할 수 있도록 백분율 또는 진행 표시 형태로 제공되는 정보 |
| 갱신 간격 | 업로드 진행률이 화면에 새롭게 반영되는 시간 간격 |
| 메타데이터 | 파일명, 크기, 업로더, 업로드 날짜, 파일 유형 등 파일에 대한 부가 정보 |
| 관리자 화면 | 사용자, 권한, 저장 공간, 공유 정책 등을 통제하는 관리용 화면 |
| RDD | Requirement Definition Document의 약자로, 요구사항 정의서를 의미한다. |
| SRS | Software Requirements Specification의 약자로, 소프트웨어 요구사항 명세서를 의미한다. |
| 유스케이스 | 사용자가 시스템을 통해 수행하는 주요 기능 또는 시나리오를 의미한다. |
| 추적성 매트릭스 | 요구사항이 어떤 유스케이스, SRS 항목, 설계 및 테스트 항목과 연결되는지 추적하기 위한 표이다. |

## 2.2 비기능적 요구사항

비기능적 요구사항은 SyncSpace가 만족해야 하는 품질 속성과 운영 기준을 정의한다. 즉, 시스템이 기능을 제공할 때 **어떤 품질 수준으로 동작해야 하는지**를 중심으로 작성한다.

### 2.2.1 운영 환경 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-001 | 시스템은 웹 브라우저 환경에서 동작해야 한다. | 상 |
| NFR-002 | 시스템은 Windows 또는 Linux 환경에서 접근 가능해야 한다. | 상 |
| NFR-003 | 시스템은 네트워크 연결 환경에서 정상적으로 동작해야 한다. | 상 |

### 2.2.2 성능 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-004 | 시스템은 일반 검색 결과를 2초 이내에 1차 표시해야 한다. | 상 |
| NFR-005 | 시스템은 파일 다운로드 요청 후 3초 이내에 전송을 시작해야 한다. | 상 |
| NFR-006 | 시스템은 동시 사용자 증가 후에도 평균 응답 시간 저하율을 20% 이내로 유지해야 한다. | 상 |
| NFR-007 | 시스템은 100MB 이상의 대용량 파일 업로드 시 업로드 진행률을 1초 이내 간격으로 갱신하여 사용자에게 표시해야 한다. 단, 네트워크 장애나 사용자의 업로드 중단 상황은 예외로 한다. | 상 |

> NFR-007은 FR-010, IR-003과 구분된다.  
> FR-010은 업로드 진행 상태를 확인할 수 있는 기능 자체를 정의하고, IR-003은 해당 기능을 사용자 화면에서 어떻게 제공할지를 정의한다.

## 2.4 요구사항 간 역할 구분

본 절은 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항 사이에서 유사하게 보일 수 있는 항목의 역할을 명확히 구분하기 위한 것이다.

특히 파일 업로드 진행 상태와 관련된 요구사항은 다음과 같이 구분한다.

| 관련 요구사항 | 구분 | 역할 | 설명 |
|---|---|---|---|
| FR-010 | 기능적 요구사항 | 기능 제공 여부 | 사용자가 업로드 진행 상태를 확인할 수 있어야 한다는 기능 자체를 정의한다. |
| IR-003 | 인터페이스 요구사항 | 화면 제공 방식 | 업로드 진행 상태를 확인할 수 있는 진행률 표시 UI가 제공되어야 함을 정의한다. |
| NFR-007 | 비기능적 요구사항 | 성능 기준 | 100MB 이상의 대용량 파일 업로드 시 진행률이 1초 이내 간격으로 갱신되어야 함을 정의한다. |

따라서 FR-010, IR-003, NFR-007은 모두 업로드 진행 상태와 관련되어 있지만 동일한 요구사항이 아니다. FR-010은 기능의 존재 여부, IR-003은 사용자에게 제공되는 인터페이스, NFR-007은 해당 기능이 만족해야 하는 성능 수준을 의미한다.

### 부록 B. 추적성 매트릭스 활용 방법

요구사항 추적성 매트릭스는 요구사항 정의서에 작성된 각 요구사항이 이후 요구사항 분석서, 소프트웨어 요구사항 명세서, 설계서, 구현 및 테스트 단계에서 누락되지 않도록 관리하기 위한 기준표이다.

본 프로젝트에서는 다음과 같은 방식으로 활용한다.

1. 요구사항 정의서의 FR, NFR, IR, OR 항목을 기준으로 유스케이스를 연결한다.
2. 각 요구사항이 SRS의 어떤 항목으로 상세화될 수 있는지 식별한다.
3. 설계 및 구현 단계에서 해당 요구사항이 실제 기능 또는 품질 속성으로 반영되었는지 확인한다.
4. 테스트 단계에서 각 요구사항의 검증 방법을 기준으로 테스트 케이스를 작성한다.
5. 요구사항 변경이 발생할 경우 관련 유스케이스, SRS 항목, 테스트 항목을 함께 갱신한다.

### 부록 C. NFR-007 수정 근거

기존 NFR-007은 “대용량 파일 업로드 시 진행 상태를 사용자에게 표시해야 한다”로 작성되어 있었다. 그러나 해당 표현은 FR-010, IR-003과 유사하게 보일 수 있다.

이에 따라 NFR-007을 다음과 같이 수정하였다.

- 기존 요구사항: 대용량 파일 업로드 시 진행 상태를 사용자에게 표시해야 한다.
- 수정 요구사항: 100MB 이상의 대용량 파일 업로드 시 업로드 진행률을 1초 이내 간격으로 갱신하여 사용자에게 표시해야 한다.

이 수정으로 인해 각 요구사항의 역할은 다음과 같이 구분된다.

| 요구사항 ID | 수정 후 역할 |
|---|---|
| FR-010 | 업로드 진행 상태를 확인할 수 있는 기능 자체를 정의 |
| IR-003 | 업로드 진행 상태를 확인할 수 있는 진행률 표시 UI를 정의 |
| NFR-007 | 대용량 업로드 상황에서 진행률 갱신 간격이라는 성능 기준을 정의 |

따라서 NFR-007은 기능 또는 UI 요구사항이 아니라, 성능 요구사항으로서의 성격이 명확해졌다.
