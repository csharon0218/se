# 프로젝트 II. 대상 시스템 품질 요소 추정서  
## SyncSpace 품질 속성 분석 및 우선순위 제안

---

## 1. 문서 목적

본 문서는 프로젝트 I에서 정의한 **SyncSpace(클라우드 파일 공유 시스템)** 를 대상으로,  
핵심 기능과 요구사항 분석 결과를 바탕으로 시스템이 반드시 갖추어야 할 품질 요소를 추정하고 정리한 문서이다.

이번 문서의 목적은 단순히 “좋은 시스템이어야 한다”는 수준이 아니라,  
**누가(이해관계자) 무엇을 기대하는지**,  
그리고 그 기대가 **어떤 품질 속성으로 연결되는지**를 구체적으로 도출하는 데 있다.

또한 본 문서는 이후 프로젝트 관리 계획서, 구현 우선순위 설정, 테스트 기준 수립에 반영될 수 있도록 작성한다.

---

## 2. 대상 시스템 요약

### 프로젝트명
클라우드 파일 공유 시스템 구축 프로젝트

### 개발 시스템명
SyncSpace

### 시스템 한 줄 정의
SyncSpace는 조직 내부 파일을 **안전하게 저장하고, 빠르게 찾고, 권한에 따라 공유할 수 있도록 지원하는 웹 기반 클라우드 파일 공유 시스템**이다.

### 핵심 기능 요약
- 파일 업로드 및 다운로드
- 폴더 생성 / 수정 / 삭제
- 파일 이동 및 정리
- 사용자 간 파일 공유
- 외부 링크 공유
- 파일 검색
- 버전 관리
- 계정 및 권한 관리
- 보안 및 접근 통제

---

## 3. 품질 요소 도출 방식

본 문서는 아래 세 가지 관점을 기준으로 품질 요소를 도출한다.

1. **사용자 관점**  
   실제 시스템을 사용하는 직원 또는 외부 협력 사용자가 기대하는 가치

2. **개발자 / 유지보수 담당자 관점**  
   시스템을 설계, 구현, 수정, 테스트, 운영 지원하는 사람이 기대하는 가치

3. **개발 조직 관리자 관점**  
   프로젝트 일정, 비용, 운영 효율, 보안, 조직 생산성을 관리하는 입장에서의 기대 가치

즉, 이번 문서는 “기능이 무엇인가”가 아니라  
**“그 기능이 어떤 품질로 제공되어야 하는가”** 에 집중한다.

## 4. 이해관계자별 기대치와 품질 요소

# 4-1. 사용자 관점: “쉽고, 빠르고, 실수해도 안전한 시스템”

사용자에게 SyncSpace는 단순한 저장소가 아니다.  
업무 중 필요한 파일을 즉시 찾고, 협업 대상과 안전하게 공유하며, 실수했을 때도 복구할 수 있어야 하는 **업무 도구**다.

### 사용자가 원하는 기대치
- 파일을 직관적으로 업로드하고 다운로드할 수 있어야 한다.
- 필요한 파일을 빠르게 검색할 수 있어야 한다.
- 폴더 구조를 통해 파일을 정리하기 쉬워야 한다.
- 다른 사용자와 파일을 안전하게 공유할 수 있어야 한다.
- 실수로 삭제하거나 잘못 수정했을 때 복구할 수 있어야 한다.
- 보안 때문에 불안하지 않아야 한다.

### 사용자 관점 핵심 품질 요소

#### 1) 사용성 (Usability)
사용자는 특별한 교육 없이도 파일 업로드, 폴더 생성, 공유 설정, 검색 기능을 쉽게 사용할 수 있어야 한다.  
버튼 위치, 화면 흐름, 정보 구조가 직관적이어야 하며, 최소한의 클릭으로 원하는 작업을 완료할 수 있어야 한다.

#### 2) 성능 (Performance)
파일 검색과 다운로드는 사용 빈도가 매우 높으므로 즉각적인 반응이 중요하다.  
느린 검색과 다운로드는 사용자의 만족도를 떨어뜨리고 업무 흐름을 끊는다.

#### 3) 신뢰성 (Reliability)
사용자는 업로드한 파일이 사라지지 않고, 삭제한 파일도 일정 기간 복구 가능하며, 이전 버전으로 되돌릴 수 있기를 기대한다.  
특히 협업 환경에서 파일 버전 관리 기능은 신뢰성의 핵심이다.

#### 4) 보안성 (Security)
공유된 파일이 허가되지 않은 사람에게 노출되지 않아야 하며, 외부 링크 공유 시에도 만료 기간과 접근 제한이 필요하다.  
사용자는 “편리하지만 위험한 시스템”보다 “안전하게 쓸 수 있는 시스템”을 더 선호한다.

### 사용자 관점 한 문장 요약
> 사용자는 **쉽고 빠르며, 실수해도 복구 가능하고, 안심하고 공유할 수 있는 시스템**을 원한다.

---

# 4-2. 개발자 / 유지보수 담당자 관점: “고치기 쉽고, 확장하기 쉬운 시스템”

개발자와 유지보수 담당자에게 중요한 것은 현재 기능보다도  
**앞으로 바뀌는 요구사항을 얼마나 잘 수용할 수 있는가** 이다.

### 개발자 / 유지보수 담당자가 원하는 기대치
- 기능 추가와 수정이 쉬워야 한다.
- 오류 발생 시 원인을 빨리 파악할 수 있어야 한다.
- 테스트하기 쉬운 구조여야 한다.
- 파일 관리, 공유, 권한, 검색 기능이 얽히지 않고 분리되어 있어야 한다.
- 향후 공동 편집, AI 검색 등 확장이 가능해야 한다.

### 개발자 / 유지보수 관점 핵심 품질 요소

#### 1) 유지보수성 (Maintainability)
정책 변경이나 기능 수정 시 전체 시스템을 크게 흔들지 않고 수정 가능해야 한다.  
예를 들어 공유 링크 만료 정책이나 버전 저장 정책 변경이 특정 모듈 수준에서 처리되면 유지보수성이 높다.

#### 2) 모듈성 (Modularity)
파일 저장, 검색, 공유, 계정, 보안 기능이 적절히 분리되어야 한다.  
모듈성이 높을수록 기능 간 영향도를 줄이고 작업 분담도 쉬워진다.

#### 3) 테스트 용이성 (Testability)
검색 정확도, 권한 설정, 파일 복구, 버전 복원 등 핵심 기능은 자동화 테스트 또는 명확한 검증 시나리오를 만들 수 있어야 한다.

#### 4) 확장성 (Extensibility)
현재 범위는 파일 공유 시스템이지만, 향후 공동 편집, 자동 분류, AI 검색 기능이 추가될 수 있다.  
따라서 처음부터 확장을 막는 구조보다는, 성장 가능한 구조가 필요하다.

#### 5) 장애 추적 가능성 (Diagnosability)
업로드 실패, 권한 오류, 만료 링크 접근, 검색 속도 저하 등 문제가 발생했을 때 원인을 추적할 수 있어야 한다.  
즉, 로그와 에러 메시지도 중요한 품질이다.

### 개발자 / 유지보수 관점 한 문장 요약
> 개발자와 유지보수 담당자는 **구조가 명확하고, 수정·테스트·확장이 쉬운 시스템**을 원한다.

---

# 4-3. 개발 조직 관리자 관점: “안정적으로 운영되고, 비용 대비 효과가 큰 시스템”

관리자에게 중요한 것은 단순한 기능 구현이 아니라  
**이 시스템이 조직의 업무 효율과 보안 수준을 실제로 얼마나 높여주는가** 이다.

### 개발 조직 관리자가 원하는 기대치
- 약 200명 규모에서도 안정적으로 운영되어야 한다.
- 파일 검색과 다운로드가 업무를 방해하지 않을 정도로 빨라야 한다.
- 계정, 권한, 저장 공간 관리가 쉬워야 한다.
- 보안 사고 가능성이 낮아야 한다.
- 향후 기능 확장 시 과도한 비용이 들지 않아야 한다.
- 시스템 도입 효과가 분명해야 한다.

### 관리자 관점 핵심 품질 요소

#### 1) 관리성 (Manageability)
관리자는 사용자 계정, 저장 공간, 권한 정책을 쉽게 관리할 수 있어야 한다.  
운영 상태를 파악하고 정책을 반영할 수 있어야 실제 조직 시스템으로서 가치가 있다.

#### 2) 보안성 (Security)
조직의 파일을 다루는 만큼 보안은 핵심 관리 지표다.  
외부 공유 기능이 있는 시스템일수록 접근 제어, 인증, 암호화는 필수다.

#### 3) 성능 및 안정성 (Performance & Stability)
약 200명 수준의 동시 사용 환경에서도 검색과 다운로드가 안정적으로 동작해야 한다.  
조직 시스템은 “가끔 느린 것”도 생산성 저하로 이어진다.

#### 4) 신뢰성 (Reliability)
파일 손실, 잘못된 권한 부여, 링크 오작동, 검색 실패 같은 문제는 관리자 입장에서 큰 리스크다.  
따라서 장애 가능성이 낮고 복구가 쉬운 시스템이어야 한다.

#### 5) 확장 가능성 (Scalability / Future-readiness)
처음에는 최소 기능으로 시작하더라도, 향후 요구사항이 추가될 수 있으므로 장기적으로 발전 가능한 구조여야 한다.

### 관리자 관점 한 문장 요약
> 관리자는 **안정적이고, 안전하며, 운영하기 쉽고, 조직 효율을 높이는 시스템**을 원한다.

## 5. 품질 요소 우선순위 매트릭스

아래는 SyncSpace에서 중요하게 다뤄야 할 품질 요소를 우선순위별로 정리한 것이다.

| 품질 요소 | 중요도 | 이유 |
|---|---|---|
| 보안성 | 매우 높음 | 조직의 중요 파일과 외부 공유 기능을 다루므로 가장 우선 |
| 성능 | 매우 높음 | 검색과 다운로드는 가장 자주 사용되는 기능이며 업무 효율과 직결 |
| 사용성 | 높음 | 사용자가 쉽게 쓰지 못하면 시스템 도입 효과가 떨어짐 |
| 신뢰성 | 높음 | 파일 손실 방지, 복구, 버전 관리가 핵심 |
| 유지보수성 | 높음 | 기능 추가와 정책 변경 가능성이 높음 |
| 관리성 | 중간 이상 | 운영자 입장에서 계정/권한/저장공간 관리 필요 |
| 확장성 | 중간 이상 | 향후 공동 편집, 자동 분류, AI 검색 기능 고려 |
| 테스트 용이성 | 중간 이상 | 핵심 기능 검증과 장애 예방에 필요 |

### 최우선 품질 목표
1. 보안성  
2. 성능  
3. 사용성  
4. 신뢰성  
5. 유지보수성

## 6. 품질 시나리오 기반 분석

이번 과제물을 조금 더 실무적으로 보이게 하기 위해, 품질 요소를 실제 상황과 연결해 정리한다.

### 시나리오 A. 사용자가 파일을 검색한다
- 상황: 사용자가 특정 프로젝트 문서를 찾으려 한다.
- 기대 품질: 성능, 사용성
- 기대 결과: 파일명 또는 조건 검색 시 빠르게 결과가 표시되어야 한다.
- 실패 시 문제: 검색 지연은 업무 흐름을 끊고 시스템 불신을 초래한다.

### 시나리오 B. 외부 협력업체와 파일을 공유한다
- 상황: 내부 직원이 외부 사용자에게 링크를 보낸다.
- 기대 품질: 보안성, 관리성
- 기대 결과: 링크 만료 기간 설정, 접근 제한, 추적 가능한 공유 방식이 필요하다.
- 실패 시 문제: 정보 유출 가능성 증가

### 시나리오 C. 사용자가 잘못 삭제한 파일을 복구한다
- 상황: 실수로 중요 파일을 삭제했다.
- 기대 품질: 신뢰성
- 기대 결과: 일정 기간 내 복구 가능해야 한다.
- 실패 시 문제: 파일 손실 및 업무 차질

### 시나리오 D. 운영 중 공유 정책이 변경된다
- 상황: 관리자가 외부 공유 정책을 강화해야 한다.
- 기대 품질: 유지보수성, 관리성
- 기대 결과: 정책 수정이 시스템 전체 개편 없이 가능해야 한다.
- 실패 시 문제: 운영 부담 증가, 수정 비용 증가

### 시나리오 E. 사용자 수가 증가한다
- 상황: 부서 확대 또는 프로젝트 증가로 사용량이 늘어난다.
- 기대 품질: 확장성, 성능
- 기대 결과: 시스템 성능 저하 없이 점진적으로 대응 가능해야 한다.
- 실패 시 문제: 서비스 불안정, 조직 불만 증가

## 7. 품질 요소별 측정 기준 제안

품질 요소를 더 설득력 있게 보이게 하려면, “좋아야 한다”에서 끝나지 않고  
가능한 한 **측정 가능하게** 적는 것이 좋다.

| 품질 요소 | 예시 측정 기준 |
|---|---|
| 사용성 | 신규 사용자가 기본 업로드/공유 작업을 짧은 안내만으로 수행 가능 |
| 성능 | 검색 결과와 다운로드 반응 속도가 사용자가 답답함을 느끼지 않는 수준 |
| 보안성 | 권한 없는 사용자는 파일 접근 불가, 외부 링크는 만료 설정 가능 |
| 신뢰성 | 삭제 파일 복구 가능, 이전 버전 조회/복원 가능 |
| 유지보수성 | 정책 변경 시 핵심 모듈 일부 수정만으로 반영 가능 |
| 관리성 | 관리자 계정/저장공간/권한 설정을 일관된 화면에서 관리 가능 |
| 확장성 | 향후 AI 검색, 공동 편집 기능 추가를 고려한 구조 유지 |

## 8. 내가 생각한 핵심 인사이트

이번 SyncSpace에서 가장 중요한 품질은 단순히 “파일을 저장할 수 있다”가 아니다.  
이 시스템의 본질은 **협업 중 발생하는 혼선과 리스크를 줄이는 것**이다.

즉, 이 시스템은 다음 세 가지 질문에 강해야 한다.

- **빠르게 찾을 수 있는가?**
- **안전하게 공유할 수 있는가?**
- **실수해도 회복할 수 있는가?**

이 세 질문이 바로 SyncSpace 품질 설계의 핵심이라고 판단한다.

## 9. 종합 결론

SyncSpace는 단순한 저장소가 아니라,  
조직 협업을 지원하는 **업무 인프라 시스템**으로 보아야 한다.

따라서 본 시스템은 기능 구현 자체보다도  
**보안성, 성능, 사용성, 신뢰성, 유지보수성**을 균형 있게 확보해야 한다.

- 사용자에게는 쉽고 빠르고 안전해야 하고,
- 개발자에게는 수정과 확장이 쉬워야 하며,
- 관리자에게는 안정적이고 통제 가능한 시스템이어야 한다.

결론적으로 SyncSpace의 품질 목표는 다음과 같이 정리할 수 있다.

> **“누구나 쉽게 쓰되, 누구나 아무렇게나 접근할 수는 없고,  
> 시스템은 빠르고 안정적으로 동작하며,  
> 이후에도 계속 성장 가능한 구조를 가져야 한다.”**
